# 보안 가이드

이 문서는 프로젝트에서 지켜야 할 보안 원칙을 정리합니다.

## 비밀 정보 관리

- API 키, DB 비밀번호 등은 절대 Git에 커밋하지 않습니다.
- `.env` 파일은 `.gitignore`에 포함되어 있습니다.
- 운영 환경에서는 환경 변수 관리 서비스(Vault, Parameter Store 등) 사용을 권장합니다.

## DB 보안

- 최소 권한 원칙:
  - 애플리케이션 계정은 필요한 테이블에 대한 최소 권한만 부여합니다.
- 운영/개발 계정 분리:
  - 개발자 로컬 계정과 운영 계정을 명확히 분리합니다.
- 접속 로그:
  - 중요한 쿼리/변경 작업에 대한 감사 로그(Audit) 설정을 검토합니다.

## API 키 관리

- Kakao, LLM 등 외부 서비스 API 키는:
  - 최소 권한 / 최소 스코프 사용.
  - 정기적으로 키 로테이션.
  - 노출 시 즉시 폐기 및 재발급.

## 네트워크/접근 제어

- DB는 외부에서 직접 접근하지 않고, 애플리케이션 서버에서만 접근하도록 네트워크를 구성합니다.
- API 서버에 대해:
  - HTTPS 사용.
  - CORS 설정은 필요한 도메인만 허용.

## 로깅과 개인정보

- 로그에 다음 정보를 남기지 않도록 주의합니다.
  - 주민번호, 전화번호, 주소 등 민감/개인 식별 정보.
  - API 키, 비밀번호, 토큰.
- 필요 시 민감 정보는 마스킹 후 저장합니다.

## 인증/인가 (현황 및 계획)

- 현재:
  - 로그인/회원 기능이 초기 버전이거나 미도입일 수 있습니다.
- 계획:
  - JWT 기반 토큰 인증 또는 세션 기반 인증 도입.
  - 역할 기반 접근 제어(RBAC)로 `/admin` 등 민감 기능을 보호.
