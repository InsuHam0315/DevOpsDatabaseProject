# DB 마이그레이션 정책

이 문서는 Oracle 스키마 변경 시 따라야 할 절차를 정의합니다.

## 기본 원칙

- 모든 스키마 변경은 SQL 스크립트로 관리.
- 변경 내역/버전/작성자/일자를 주석으로 기록.
- 운영/스테이징/개발 환경의 스키마를 최대한 일치시키기.

## 스크립트 네이밍 예시

- `V001__init_schema.sql`.
- `V002__add_run_summary_indexes.sql`.
- `V003__alter_assignments_add_status.sql`.

## 적용 절차

1. 새로운 변경 스크립트 작성.
2. 개발 환경에서 테스트.
3. 스테이징 환경에 적용 후 애플리케이션 테스트.
4. 운영 환경에 적용.

## 롤백 전략

- 가능한 한 다운그레이드 스크립트를 함께 작성.
- 위험도가 높은 변경(컬럼 삭제, 타입 변경)은 별도 검토 및 백업 후 수행.
